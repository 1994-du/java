## 八、WebSocket接口

**连接地址**: `ws://服务器地址:1234/ws`

### 前端发送的消息类型

| type | 说明 | payload字段 |
|------|------|-------------|
| userOnline | 用户上线 | userId: 用户ID<br>username: 用户名 |
| userOffline | 用户下线 | userId: 用户ID<br>username: 用户名 |
| chat | 群聊消息 | userId: 用户ID<br>username: 用户名<br>message: 消息内容<br>image: 图片Base64（可选） |
| card | 卡片消息 | userId: 用户ID<br>username: 用户名<br>title: 卡片标题<br>description: 卡片描述<br>url: 卡片链接<br>imageUrl: 卡片图片（可选） |

### 后端返回的消息类型

| type | 说明 | payload字段 | 触发时机 |
|------|------|-------------|----------|
| onlineUsers | 在线用户列表 | `[{userId, username, avatar, time}]` | 连接成功时自动返回 |
| chatHistory | 最近一天聊天记录 | `[{id, userId, username, message, avatar, time}]` | 连接成功时自动返回 |
| userOnline | 用户上线通知 | userId, username, time, onlineCount | 有用户发送上线消息时广播 |
| userOffline | 用户下线通知 | userId, username, time, onlineCount | 有用户发送下线消息时广播 |
| chat | 群聊消息广播 | userId, username, message, time, avatar, imageUrl | 有用户发送聊天消息时广播 |
| card | 卡片消息广播 | userId, username, title, description, url, imageUrl, time, avatar | 有用户发送卡片消息时广播 |
| cardSuccess | 卡片发送成功 | message, cardTitle | 卡片发送成功后返回给发送者 |
| error | 错误消息 | message | 发生错误时返回 |

### WebSocket消息示例

**用户上线：**
```json
{
  "type": "userOnline",
  "payload": {
    "userId": 14,
    "username": "admin"
  }
}
```

**发送群聊消息：**
```json
{
  "type": "chat",
  "payload": {
    "userId": 14,
    "username": "admin",
    "message": "大家好"
  }
}
```

**发送卡片消息：**
```json
{
  "type": "card",
  "payload": {
    "userId": 14,
    "username": "admin",
    "title": "分享链接",
    "description": "这是一个有趣的网站",
    "url": "https://example.com",
    "imageUrl": "https://example.com/image.jpg"
  }
}
```

**用户下线：**
```json
{
  "type": "userOffline",
  "payload": {
    "userId": 14,
    "username": "admin"
  }
}
```

### 聊天记录存储

- 聊天记录存储在 `sys_chat_messages` 表中
- 只保留最近一天的聊天记录
- 每小时自动清理超过一天的记录
- 用户连接WebSocket时自动获取最近一天的聊天记录

## 通用响应格式

所有接口统一使用 ApiResponse 格式响应：

```json
{
  "success": true,
  "message": "操作成功",
  "data": { ... },
  "total": 100
}
```

## 认证说明

- 需要认证的接口通过 Cookie 中的 `token` 或请求头 `Authorization: Bearer xxx` 进行身份验证
- 登录成功后会返回用户信息并设置token cookie
- WebSocket连接后需要发送 `userOnline` 消息进行身份注册

## 文件上传说明

- 文件上传限制最大5MB
- 支持的文件格式：jpg、jpeg、png、gif、pdf、doc、docx、txt
- 头像上传仅支持图片格式
